## 1. ひと言で言う Tokiq

Tokiq は、リアルタイムの多人数参加型で **スキルベース** のタイミングゲームです。目標タイム
（例：`10.00 秒`）が表示され、タイマーがカウントアップし、各プレイヤーは目標に達したと思った
瞬間に **STOP** をタップします。**目標タイムに最も近かったプレイヤーが、そのラウンドの賞金
プールを獲得します。** プレイヤーは少額の参加費を支払い、プラットフォームはハウス手数料を
受け取ります。Aviator のような「クラッシュ系」ゲームと同じ系統に属しますが、フックは上昇する
倍率ではなく、精度と度胸（「あと少しだった」という感覚）にあります。**現時点で手元にあるのは、
説得力のあるフロントエンドのみのプロトタイプです。** すなわち、本物のマルチプレイヤー、決済、
不正対策はまだありません。本書はその事実に正直に向き合い、取り組みを正しく位置づけます。難しい
のはタイマーをつくることではありません。難しいのは、プレイヤーが **また戻ってきて、信頼し、お金
を使い、友人を誘う** ことを証明することです。

## 2. Tokiq のようなプロダクトの考え方

> この章は、以降すべての土台となる **考え方** です。成功するスタートアップが実際にどう築かれて
> いるかから導いた、平易な短いレッスンであり、渡辺さんが私たちに促してきた思考（機能の積み重ね
> ではなく、まず顧客、次に価値、次に収益、次に持続可能性）とほぼ完全に一致します。

### 2.1 唯一にして最大の考え（すべてはここに戻る）

> ### 💡 **解決策（タイマー）に恋をしてはいけません。課題に恋をしてください。**

多くのチームは一直線に進みたくなりますが、まさにその直線こそ、ほとんどの製品が失敗する道です。
実証された道筋は、プロダクトを **後ろ** に置き、人を理解した後に初めてつくります。

```
   ✗ 陥りやすい道筋
        アイデア  →  開発  →  ローンチ                （多くの製品が失敗する道）

   ✓ 実証された道筋
        顧客 → 課題 → 価値 → 市場 → ビジネスモデル → プロダクト → 開発 → 成長 → スケール
        └──── まず「人」を理解する ────┘        └──── つくるのは「その後」 ────┘
```

つまり、最初に重要となる問いは、まったく技術的なものではありません。

```
   なぜ人はお金を払ってまで遊ぶのか？
            ↓
   なぜ明日もまた戻ってくるのか？
            ↓
   なぜ友人を誘うのか？
            ↓
   なぜ投資家が関心を持つのか？
```

**Tokiq が売っているのはタイマーではありません。感情を売っています。** すなわち、期待、緊張、
希望、惜敗、勝利の瞬間、そして社会的な承認です。Aviator において飛行機は商品ではなく、感情こそ
が商品です。私たちの STOP ボタンでも同じことが言えます。

### 2.2 学ぶべき順序（5 つのフェーズ）

チームのためのシンプルなはしごです。順番に登っていきます。

| フェーズ | 焦点 | 答える問い |
|:---:|---|---|
| **1. プロダクト思考** | 顧客・価値・収益・コスト・成長・投資家・持続可能性・システム思考 | *なぜこれはビジネスとして存在すべきか？* |
| **2. 市場調査** | Aviator・Crash・Mines・Plinko を誰が遊ぶか、なぜ、いくら使うか、何を好み何を嫌うか | *本当に「お金を払う層」がいるか？* |
| **3. 心理** ⭐ *(Tokiq に最も重要)* | 期待、緊張、希望、惜敗、社会的証明、勝利の瞬間 | *本当に売っている「感情」は何か？* |
| **4. ゲーム経済** | 参加費、賞金プール、勝者、プラットフォーム取り分、運営、成長 | *お金のループは自走できるか？* |
| **5. 投資家の視点** | 課題、解決策、なぜ機能するか、リスク（不正、遅延、単調さ、易しすぎ）、機会（リーダーボード、トーナメント、チーム、ジャックポット、ライブ配信） | *外部の本気の相手が出資するか？* |

### 2.3 プロダクト・マーケット・フィット（PMF）への 7 ステップ（平易な言葉で）

**プロダクト・マーケット・フィット（PMF）** とは、十分な数の人々がそれを強く欲しがり、繰り返し
戻ってきて、友人に伝えてしまう状態のことです。以下は広く使われる 7 ステップを日常語に置き換え、
各ステップが **Tokiq** にとって何を意味するかを添えたものです。

| # | ステップ | シンプルな教訓 | Tokiq にとっての意味 |
|:---:|---|---|---|
| **1** | **PMF を理解する** | 製品が市場をつくるのではなく、**市場が製品を引き寄せます**。良い製品でも悪い市場では負けます。フィットの最良の兆候は売上ではなく、**自然な口コミ** です。 | 「タイマーは面白いか？」ではなく「人が繰り返し戻りたくなるほど魅力的か？」と問う。 |
| **2** | **PMF を測る** | 本当の試金石はダウンロード数ではなく **継続率** です。**Sean Ellis テスト**：「これが使えなくなったらどう感じるか？」に **40% 以上** が「非常に残念」と答えれば、フィットの可能性が高い。 | インストール 100 万でも誰も戻らなければ失敗。**毎日遊ぶ 100 人こそ「生きている」。** |
| **3** | **顧客開発** | **つくる前に話を聞く。** **ジョブ理論（Jobs-To-Be-Done）**：人は製品を買うのではなく、ある成果のために製品を「雇う」。ヘビーユーザー、ライトユーザー、そして **離脱ユーザー**（最も学びが多い）に話を聞く。 | プレイヤーは「タイミングの正確さ」のために Tokiq を雇うのではありません。**興奮、ステータス、希望、競争、ストレス発散** のために雇うのです。 |
| **4** | **反復（イテレーション）** | **検証する前につくらない。** まず **最も安価な実験**（モックアップ、ランディングページ）から。そして「つくる → 測る → 学ぶ → 繰り返す」。**アウトプットではなくアウトカム** を測る（コード行数ではなく継続率）。 | v1 はタイマー、次にフィードバック、ランキング追加、トーナメント追加、と常に学び続ける。 |
| **5** | **継続とエンゲージメント** | **「アハ体験」** を見つける。**フックモデル** を使う（§5 参照）。**予測できない** 報酬は、**大きい** 報酬より強い習慣をつくる。 | アハ体験は初の惜しいストップ、初勝利、初のリーダーボード入りなど。変動報酬は「あと少しだった」という惜敗。 |
| **6** | **成長チャネル** | 10 個のチャネルを追うのではなく、**まず効く 1 つを見つける**。ゲームでは最良のチャネルは有料広告ではなく、紹介、コミュニティ、配信者、バズる動画であることが多い。製品が **自ら広がる** よう設計する。 | **他人が勝つのを見ること自体が、組み込み型の成長エンジン**（社会的証明）。 |
| **7** | **スケール** | **フィットの前に拡大しない。** それは「まだ着いていない火にガソリンを注ぐ」ようなもの。正しい順序は **継続 → 獲得 → 組織 → 市場**。 | プレイヤーが戻り、友人を連れてくると証明した **後にのみ**、ゲームを大きくつくる。 |

旅全体を 1 つのはしごとして示します。多くのチームは一番上の段（スケール）にいきなり飛びつき
ます。正しいやり方は一番下の段（プレイヤーの理解）から始め、一段ずつ登ります。

```
            ┌──────────────────────────────────────────────┐
   誤った ▲ │  7 ▸ スケール                                  │
   出発点 │ │  6 ▸ 成長                                      │
         │ │  5 ▸ 継続とエンゲージメント                     │
         │ │  4 ▸ 反復                                       │
         │ │  3 ▸ 顧客開発                                   │
         │ │  2 ▸ PMF を測る                                 │
   正しい │ │  1 ▸ プレイヤーと課題を理解する                  │
   出発点 ▼ └──────────────────────────────────────────────┘
```

### 2.4 「リサーチボード」：壁に貼っておく問い

チームの焦点を保つ最速の方法は、実際に答えようとしている問いを貼り出すことです。まずは次の
**5 つの中核的な問い** から。

```
   1 ▸ 人は「実際に」なぜ遊ぶのか？（言うことではなく、することを見る）
   2 ▸ なぜ戻ってくるのか？
   3 ▸ 何が「社会的」な関与を生むのか？
   4 ▸ 何が「信頼」を生むのか？（Aviator の真の強み＝「証明可能な公正性」）
   5 ▸ システムはどう経済的に自走するのか？
```

より深いセッション向けの **10 問版**：

> なぜ遊ぶか · なぜ戻るか · なぜ友人を誘うか · なぜお金を使うか · なぜ Aviator ではなく
> **Tokiq** を選ぶか · 売っている感情は何か · 何が信頼を築くか · 何が **勝利を記憶に残す** か ·
> 何が **敗北を許容可能に** するか · 何が長期の継続を生むか。

## 3. これまでに何をつくったか（プロトタイプの現状）

このプロトタイプは **フロントエンドのみの MVP** であり、デモとコンセプト検証のために、ライブ
な多人数プレイ製品に「見え・感じられる」よう作られています。本物のゲームではなく、それは意図的
な設計です。

**ライブプロトタイプ:** https://slide-desk.vercel.app/

> シンプルに：ブラウザで動く現代的な Web アプリ（React + Vite + Tailwind）で、背後にサーバーは
> ありません。ゲーム全体は 4 つの画面を流れます。

```
   ロビー  →  カウントダウン(3·2·1)  →  進行中(STOP をタップ)  →  結果(勝者)  →  次のラウンド
```

### 3.1 本物 と 演出 の違い

| ✅ 本物（実際に動くコード） | 🎭 シミュレーション（端末側の演出） |
|---|---|
| タイマーとあなたの STOP タップ | 他プレイヤーのストップタイム（目標付近に分布） |
| あなたの精度と勝者判定（最も近い人が勝ち） | ライブチャット |
| ラウンドの賞金計算とあなたの残高 | 勝者・アクティビティのフィード |
| あなたの戦績、連勝、ベスト精度 | メガジャックポットの増加 |
| | リーダーボードの順位 |

> **真にインタラクティブなのは、タイマーとあなたのタップだけです。** これは意図的です。「あと
> 少しだった」という感覚を証明するために、まさに必要なフックだからです。それ以外はすべて、画面
> を生き生きと見せるための演出です。

その他の意図的なデモ上の選択：
- テーブルは少人数（**8〜14 人**）に保ち、デモ中に人間が現実的に勝てるようにしています。実際の
  数百人規模では勝つことはほぼ不可能になります。
- 対戦相手のタイムは **目標付近のベル型ノイズ** を用い、混沌ではなくスキルらしく見えます。
- 作り込み：目標に近づくと締まるストップウォッチ音、タイマー作動中のみ流れる音楽、紙吹雪と勝利
  音、3 秒のイントロ、アプリとしてインストール可能（PWA）、完全なレスポンシブ、そして **3 言語
  （English / Español / 日本語）**。

### 3.2 現在の（プロトタイプの）お金のモデル ※説明用のみ

```
   参加費       :  ¥100
   ハウス率     :  10%
   賞金プール   =  人数 × 参加費 × (1 − ハウス率)

   例：10 人の場合
        10 × ¥100 × 0.9  =  ¥900  ──▶ 勝者
                            ¥100  ──▶ プラットフォーム
```

これは標準的な「プール＆レーキ（手数料）」構造です。**仮置きの値であり、検証済みではありません。**
プレイヤーが実際にどの参加費・プール規模・ハウス率を受け入れるかは、まだ検証していません。

### 3.3 まだ「あえて」つくっていないもの（本当に難しい部分）

これらはプロトタイプの **範囲外** に意図的に置いており、本物の難所です。

1. **本物のマルチプレイヤー：** ネットワーク通信、マッチング、全員のラウンドを同一の瞬間に開始
   すること。
2. **不正対策／公正性：** 現在、ストップタイムは **プレイヤーの端末側** で計測しています。本番では
   **サーバー側で権威ある計測** と遅延補償が必要です。*これが中核の技術課題です。*
3. **決済と規制：** 参加費 ＋ 賞金プール ＋ ハウス手数料は、**国ごとの法的レビュー（特に日本）** が
   必要です。スキルベースという位置づけは賭博性の論点に有利ですが、**自動的に適用除外になるわけ
   ではありません。**
4. **ラウンドの勝利からメガジャックポットへの本当の道筋**（現状は理想を示す数字にすぎません）。

> **これが計画にとって重要な理由。** 対戦相手がシミュレーションであるため、プロトタイプは **ループ
> の手触り** と **惜敗のフック** は検証できますが、**真のマルチプレイヤー継続率や実際の支払い意欲は
> まだ検証できません。** 文書 2 はまさにこの制約を前提に組み立てています。

## 4. 市場と競合の状況

> ⚠️ **出典についての注記。** 以下の数値は以前の机上調査（Aviator / Spribe 分析）に基づきます。
> **方向性を示すものであり、社外・投資家向けに用いる前に独自に再検証が必要** です。「現時点で私たち
> が信じている内容」としてお取り扱いください。

### 4.1 ベンチマーク：Spribe と Aviator

- **会社：** Spribe。**2018 年** に **David Natroshvili 氏**（ジョージアの元経済担当第一副大臣）が
  創業。本拠地はトビリシ、ワルシャワとキーウにオフィス。
- **規模：** Aviator は月間 **6,000 万人以上** のプレイヤー、クラッシュ系カテゴリの約 **92%** の
  シェアを持つと報じられています。
- **モデル：** **B2B サプライヤー**。自らカジノは運営せず、数千の事業者プラットフォームに Aviator を
  **ライセンス供与**（報じられる数字は約 4,500〜6,000 以上）。
- **技術：** **RTP（還元率）97%**、軽量で **モバイルファースト**、**毎秒 2,000 件以上** のベットを処理。
- **信頼（核心の機能）：** **「証明可能な公正性」**。暗号方式（SHA-256）とプレイヤー由来のデータに
  より、事業者も開発者もラウンドを操作できず、プレイヤーは後から結果を検証できます。
- **主流化：** **UFC** や **AC ミラン** との提携により、ブランドを「暗がりの賭博」から社会的な
  エンターテインメントへと押し上げました。

### 4.2 このカテゴリが重要な理由

- **アフリカ：** 近年 **約 53% 成長** と報じられ、若年でモバイル中心の層が、従来のスロットより
  速くて社会的なセッションを好みます。
- **事業者の経済性：** 高回転により **ゲーム収益が 10% 以上向上** との報告。
- **参入障壁の低さ：** 価値は重厚な 3D ではなく **数理と社会的設計** にあり、開発コストは低く、
  リターンの可能性は高い。
- **市場規模：** **1,000 億ドル超** の iGaming エコシステムの一部。

### 4.3 研究すべき比較対象

**Aviator · Crash · Mines · Plinko · Lucky Jet · JetX · Spaceman。**
それぞれについて、誰が・なぜ・年齢・支出額・好きな点・嫌いな点 を学びます。

### 4.4 Tokiq の共通点と相違点

| | **Aviator（クラッシュ系）** | **Tokiq（タイミング系）** |
|---|---|---|
| 中核の操作 | 飛行機が墜落する前にキャッシュアウト | 目標タイムに最も近く STOP をタップ |
| 緊張の源 | 上昇する倍率、欲と恐怖 | プレッシャー下の精度、惜敗 |
| 結果の感触 | 運主導（キャッシュアウトの技あり） | **スキル主導**（最も近い人がプールを獲得） |
| 信頼の仕組み | 証明可能な公正性（SHA-256） | **設計に組み込む必要あり**（証明可能な公正性） |
| 社会的レイヤー | ライブのベットとチャット | ライブフィード、チャット、リーダーボード（予定） |

> **未解決の戦略的問い。** *なぜプレイヤーは Aviator ではなく Tokiq を選ぶのか？* まだ検証済みの
> 答えはありません。有力な候補は、より強い **スキル感**（「自分は上達できる」）と、**より清潔で社会
> 的、カジノ臭の少ない** 体験ですが、これらは **仮説であって事実ではありません。** 文書 2 はこれを
> 検証するために設計されています。

## 5. プレイヤー心理：本当に遊びを駆動するもの

人がこの種のゲームに戻ってくるのは、報酬が **大きい** からではなく、**予測できない** からです。
私たちが働いていると考える力学は次のとおりです。

- **期待と緊張：** 目標に向かうカウントアップ。
- **惜敗：** 「0.03 秒差」はきれいな敗北よりも人を動かします。「次こそ」とささやくのです。
- **変動報酬：** 不確実な結果は、固定の報酬より強い習慣をつくります。
- **社会的証明：** 他人が勝つのを見ると、遊びたく（そして共有したく）なります。
- **希望とステータス：** リーダーボード、連勝、承認。
- **スキル感：** 上達できると信じることが反復プレイを駆動し、スキルベースという位置づけを支えます。

これらすべては、自己強化のループである **フックモデル** に対応します。

```
        ┌──────────────────────────────────────────────────────┐
        │                                                        │
        ▼                                                        │
     トリガー ──▶ 行動 ──▶ 変動報酬 ──▶ 投資 ──────────────────────┘
   (通知)      (STOP)    (どれだけ近い?)  (残高・連勝・順位)
```

ループは毎回締まっていきます。あらゆる「投資」（連勝、順位、残高）が、次に戻ってくるための
トリガーになるのです。

## 6. 生きたシステムとしてのビジネス

ゲームは単なるプレイ体験ではありません。**自らを養う経済システム** です。

```
        ┌──────────── 生み出した価値を再投資する ────────────┐
        │                                                    │
   プレイヤー ──▶ 参加費 ──▶ 賞金プール ──▶ 勝者への支払い      │
                    │                                        │
                    └──▶ プラットフォーム取り分 ──▶ 運営・マーケ ┘
                                              (さらに多くのプレイヤーを獲得)
```

いずれは 3 つの問いに答える必要があります。*スケールするか？ 利益が出るか？ 新規プレイヤー獲得
コストを賄えるか？* ただし、これらにはまだ答えません。プレイヤーが欲しがり、戻ってくると証明した
**後** の話です。

## 7. 主要なリスクと未解決の問い

| 領域 | リスク／未解決の問い |
|---|---|
| **エンゲージメント** | ループは易しすぎ・単調すぎ・時間とともに退屈ではないか？ |
| **継続** | 昨日負けたプレイヤーは今日も戻ってくるか？（成否を分ける問い） |
| **差別化** | なぜ Aviator や他のクラッシュ系ではなく Tokiq なのか？ |
| **公正性／不正** | 端末側計測は悪用可能。サーバー側の権威ある計測と遅延処理が必要。 |
| **信頼** | プレイヤーが信じられる「証明可能な公正性」を提供できるか？ |
| **支払い意欲** | プレイヤーは実際にどの参加費・プール・ハウス率を受け入れるか？ |
| **規制** | 参加費 ＋ 賞金プールは市場ごと（**特に日本**）の法的レビューを要する。スキルの位置づけは助けになるが自動的な適用除外ではない。 |
| **経済性** | ハウス取り分は、実規模で運営費 ＋ 獲得コストを賄えるか？ |

## 8. 唯一にして最大の教訓（私たちの北極星）

> ### 🧭 解決策（タイマー）に恋をしてはいけません。課題に恋をしてください。
